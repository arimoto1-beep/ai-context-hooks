# 例と使い方

---

## サンプル会議ログ

→ [examples/raw/2026-06-14_meeting.md](../examples/raw/2026-06-14_meeting.md)

CSV出力機能について話し合った会議の短いログです。

含まれているニュアンス：

- CSV出力は今回はやらない方向
- ただし不要と確定したわけではない
- 将来的には欲しいと言われそう
- 設計上、出力形式を増やしやすいか少し気になる
- 今回は時間の都合で対象外にする

---

## 悪いメモの例

```text
CSV出力は将来対応予定。出力形式は拡張可能な設計にする。
```

### なぜ悪いか

1. 「将来対応予定」は決定事項ではない。「いつかやるかもしれない」という話が「予定」に変わっている。
2. 「出力形式は拡張可能な設計にする」は設計方針として断言されているが、元の会話では「気になる」「確認したい」程度の懸念として出てきた話。
3. 原文参照がない。この文章だけを見たAIは「CSV出力の実装が予定されており、拡張可能な設計が決まっている」と解釈しかねない。

---

## 良い索引メモの例

→ [examples/hooks/ctx-20260614-001.md](../examples/hooks/ctx-20260614-001.md)

```markdown
## ctx-20260614-001

- **id**: ctx-20260614-001
- **source**: examples/raw/2026-06-14_meeting.md（00:12:01〜00:13:30）
- **topic**: CSV出力機能の扱い
- **labels**: 今回対象外 / 将来要望の可能性 / 設計懸念 / 未決
- **confidence**: low
- **hook_note**:
  CSV出力については今回は対象外という方向の会話。
  ただし、不要と確定したわけではなく、将来の要望として出てくる可能性が話題に上がっている。
  出力形式の拡張性についても少し言及があったが、設計方針として決まったわけではない。
- **ai_usage**: 仕様として直接採用しない。この話題が関係する場合は原文 00:12:01〜00:13:30 を確認すること。
- **cautions**: 「今回対象外」を「永続的に不要」と解釈しないこと。設計懸念は方針ではない。
```

### 何が良いか

1. `labels` が複数あり、性質を区別している（対象外・将来要望・設計懸念・未決）
2. `confidence: low` で確からしさを正直に示している
3. `hook_note` が断定を避け、「方向の会話」「可能性が話題に上がっている」という温度感を保っている
4. `source` に原文ファイルと時刻範囲が記載されている
5. `ai_usage` でAIへの扱い方を明確にしている
6. `cautions` で誤用しやすいポイントを明示している

## AIにどう使わせるかの例

索引メモを渡してAIに確認してもらう場合、`prompts/use_context_hook.md` のプロンプトを使います。

### 良い使い方

```text
以下の索引メモを参照して、CSV出力に関する設計検討で注意すべきことを教えてください。
ただし、索引メモだけで結論を出さず、必要に応じて原文（source に記載）を確認するよう進めてください。
```

→ AIは索引メモを「手がかり」として使い、必要なら原文を読みに行く

### 悪い使い方

```text
CSV出力の仕様を索引メモから整理してください。
```

→ 索引メモを仕様書として扱わせてしまっている。未決・懸念・対象外が「仕様」に変わるリスクがある。

---

## レビュー時に確認する観点

`prompts/review_context_hook.md` に詳細なレビュー観点があります。

主なチェックポイント：

| 観点 | 確認内容 |
|------|----------|
| 断定の有無 | 原文にないことを断定して書いていないか |
| 未決の扱い | 未決事項を決定事項として書いていないか |
| 懸念の扱い | 懸念・心配を仕様・方針として書いていないか |
| 原文参照 | source に原文ファイルと参照箇所が記載されているか |
| labels の適切さ | 性質を正しく分類できているか |
| confidence | 高すぎる設定になっていないか |
| ai_usage | 誤用防止の指示が明確か |
| 単体誤用リスク | この索引メモだけを見てAIが誤判断しないか |
